## 3. デプロイ後に初めて検証される「未踏の領域」
デプロイ失敗時、以下の API 連携や環境依存の箇所を優先して確認してください。これらはテスト時に **モック (Mock)** されており、実環境特有の制約によりエラーが発生する可能性があります。

### A. 認証・認可 (Authentication & IAM)
- **Secret Manager**: シークレット名 (`JOBCAN_PASSWORD_*`) の不一致や、サービスアカウントの参照権限不足。
- **Google Calendar API**: ドメイン全体の委任 (Domain-Wide Delegation) が未完了。
- **Slack API**: ボットトークン (`xoxb-...`) の有効期限切れやスコープ不足。

### B. インフラ・ネットワーク (Infra & Networking)
- **Vertex AI (Gemini 3)**: 対象プロジェクトでの API 有効化状態、およびリージョン (`us-central1` vs `global`)。
- **Cloud Tasks**: キューの作成状態と、Worker (Cloud Run) への認証済みリクエスト。
- **Playwright (Jobcan)**: ジョブカン側の UI 仕様変更（クラス名の変更等）や、ヘッドレスブラウザ対策によるブロック。

## 4. 異常発生時の切り分けフロー
1. **`make logs` を確認**:
   - `403 Forbidden` → IAM 権限または API 有効化を疑う。
   - `Timeout` → 外部サイト（ジョブカン）の応答遅延を疑う。
2. **`make secrets` を確認**:
   - 必要なシークレットが定義されているか、命名規則が正しいかを確認。
3. **ローカル再検証**:
   - `make test` を実行。ここで落ちる場合はロジックのバグ、通る場合は環境設定（GCP/Slack等）の問題。
